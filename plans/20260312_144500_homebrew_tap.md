# Plan: Homebrew Tap for bab

## Goal

Let users install bab via `brew install babmcp/tap/bab`.

## Decision: Formula in a separate tap repo

- **Separate repo**: `git@github.com:babmcp/homebrew-tap.git`
- **Formula, not Cask** — per [Homebrew docs](https://docs.brew.sh/Acceptable-Casks): "open-source and CLI-only" apps must be Formulae, not Casks
- Users: `brew tap babmcp/tap && brew install bab` or shorthand `brew install babmcp/tap/bab`
- Updates: `brew upgrade bab`
- **IMPORTANT**: No homebrew-related files in the `babmcp/bab` repo — everything lives in `babmcp/homebrew-tap`
- This is a tap formula (not homebrew-core). `brew audit --strict` core rules do not apply to taps. Pre-built binary downloads are fine.

## Formula design

```ruby
class Bab < Formula
  desc "Bab MCP server — AI-assisted development tools"
  homepage "https://github.com/babmcp/bab"
  version "X.Y.Z"

  on_macos do
    if Hardware::CPU.arm?
      url "https://github.com/babmcp/bab/releases/download/vX.Y.Z/bab-darwin-arm64"
      sha256 "<hash>"
    else
      url "https://github.com/babmcp/bab/releases/download/vX.Y.Z/bab-darwin-x64"
      sha256 "<hash>"
    end
  end

  on_linux do
    if Hardware::CPU.arm?
      url "https://github.com/babmcp/bab/releases/download/vX.Y.Z/bab-linux-arm64"
      sha256 "<hash>"
    else
      url "https://github.com/babmcp/bab/releases/download/vX.Y.Z/bab-linux-x64"
      sha256 "<hash>"
    end
  end

  # Pre-built binary — no compilation needed
  bottle :unneeded

  def install
    # Homebrew downloads one URL per platform; the file is the binary itself
    # Hardcode expected name to avoid fragile Dir.glob
    platform = OS.mac? ? "darwin" : "linux"
    arch = Hardware::CPU.arm? ? "arm64" : "x64"
    bin.install "bab-#{platform}-#{arch}" => "bab"
  end

  test do
    assert_match version.to_s, shell_output("#{bin}/bab --version")
  end
end
```

## Auto-update on release

Decoupled via `repository_dispatch` — each repo owns its own logic:

1. `babmcp/bab` release workflow publishes binaries + `checksums.sha256`
2. Post-release job sends `repository_dispatch` event to `babmcp/homebrew-tap` with payload: `{ version, checksums: { darwin-arm64: "...", ... } }`
3. `babmcp/homebrew-tap` has its own workflow that listens for `repository_dispatch`, rewrites `Formula/bab.rb` with new version + hashes, commits + pushes
4. **Auth**: use a fine-grained PAT scoped ONLY to `babmcp/homebrew-tap` (contents: write). Do NOT use a broad `repo`-scoped PAT.
5. **Guard**: the dispatch only fires if ALL build matrix jobs succeed. If a platform build fails, no formula update happens (prevents broken checksums).
