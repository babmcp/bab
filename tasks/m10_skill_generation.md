## Phase 1: Core Module

### M10-T01: Agent discovery + staleness + writer
- [x] Create `src/skills/index.ts` — public API: `regenerateSkills(config, options?)`
- [x] Signature: `regenerateSkills(config: BabConfig, options?: { force?: boolean; agent?: string; stderr?: WritableLike }): Promise<void>`
  - `config` is the only required param — everything else has sensible defaults
  - `force` bypasses staleness (for `bab onboard`)
  - `agent` targets a specific agent (for `--agent` flag)
  - `stderr` defaults to `process.stderr`
- [x] Agent discovery: check config dirs (`~/.claude/`, `~/.codex/`)
  - Verify OpenCode skills dir before adding (may be `~/.config/opencode/` not `~/.opencode/`)
- [x] Only detect agents whose config dir exists (do NOT create it)
- [x] Create `skills/` subdir if parent agent dir exists but skills dir doesn't
- [x] Staleness check: fingerprint = SHA-256 of `${bab_version}:${sorted_plugin_ids}:${sorted_tool_names}`
- [x] Read `.bab-skills.json` for comparison, single string fingerprint match
- [x] Missing metadata → stale
- [x] User-modified detection: skip SKILL.md lacking `<!-- Auto-generated by bab v` header
- [x] Warn to stderr when skipping user-modified files
- [x] Atomic directory replacement:
  1. Build skill dir in temp location: `<skills-dir>/.bab-staging-<nonce>/`
  2. Write SKILL.md + references/ + .bab-skills.json into staging
  3. If existing `bab/` dir: rename to `<skills-dir>/.bab-old-<nonce>/`
  4. Rename staging → `bab/`
  5. Delete old dir
  6. On failure: restore old dir if it was moved, clean up staging
